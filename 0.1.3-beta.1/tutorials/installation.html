Installation#

Development with Dev Containers#

The easiest way to get started is using a Dev Container.

Prerequisites#

  • Docker installed and running

  • VS Code with the Dev Containers extension

Steps#

  1. Clone the repository and open it in VS Code.

  2. VS Code will prompt “Reopen in Container” — click it.

  3. Wait for the container to build. The postCreate script will install Node.js, npm dependencies, Python dependencies, and pre-commit hooks.

  4. Copy .env.example to .env and fill in the required values:

    cp .env.example .env

  5. Start the dev server:

    npm run dev

  6. The app is available at http://localhost:5173.

Running Checks with Tox#

Tox is used as a task runner for linting and documentation builds. Run all default checks in parallel with:

uv run tox -p

Available Environments#

Environment

Command

Description

pre-commit

uv run tox -e pre-commit

Run all pre-commit hooks on all files

docs

uv run tox -e docs

Build Sphinx documentation

docs-autobuild

uv run tox -e docs-autobuild

Live-rebuild docs on file changes

Run a specific environment:

uv run tox -e docs

Run multiple environments:

uv run tox -e pre-commit,docs

Production Deployment#

Install from OCI registry#

The Helm chart is published to the GitHub Container Registry on each tagged release. Install directly from the OCI registry:

helm install argocd-monitor oci://ghcr.io/epics-containers/charts/argocd-monitor \
  --set argocd.url=https://argocd.example.com \
  --set ingress.enabled=true \
  --set ingress.host=argocd-monitor.example.com

Install from local chart#

Alternatively, install from the source tree:

helm install argocd-monitor helm/argocd-monitor \
  --set argocd.url=https://argocd.example.com \
  --set ingress.enabled=true \
  --set ingress.host=argocd-monitor.example.com

See helm/argocd-monitor/values.yaml for the full list of configurable values.
